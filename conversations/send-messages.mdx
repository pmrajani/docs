## Send a text message

1. Open a conversation from the inbox.
2. Click the message input box at the bottom.
3. Type your message and press **Enter** or click **Send**.

You can use line breaks with **Shift + Enter**.

<Warning>
  Free-form messages are only available within 24 hours of the customer's last message. After that window closes, use **Send template** instead. See [using templates](/conversations/using-templates).
</Warning>

## Send media

WapNotify supports sending images, documents, audio files, and video.

1. Click the **attachment icon** (paperclip) in the message input bar.
2. Select the file type you want to send: image, document, audio, or video.
3. Choose the file from your device.
4. Add an optional caption (for images and documents).
5. Click **Send**.

**File limits (Meta-enforced):**

| Type | Max size |
|---|---|
| Image (JPG, PNG) | 5 MB |
| Document (PDF, DOCX, etc.) | 100 MB |
| Audio (MP3, OGG, AAC) | 16 MB |
| Video (MP4) | 16 MB |

## Use quick replies

Quick replies are pre-written message snippets that save typing time for common responses.

1. Type `/` in the message box to open the quick reply picker.
2. Start typing the title of the quick reply to filter the list.
3. Click a quick reply to insert it into the message box.
4. Edit the text if needed, then send.

Admins can manage the quick reply library at **Settings → Quick Replies**.

## Edit a sent message

You can edit a text message after sending it, as long as WhatsApp allows edits for that message type.

1. Hover over the message you want to edit.
2. Click the **Edit** icon (pencil).
3. Update the text in the input field.
4. Press **Enter** to save.

An "edited" label appears on the message after saving. You cannot edit media messages.

## Message statuses

Each outbound message shows a delivery status indicator:

| Status | Meaning |
|---|---|
| Sent | Delivered to WhatsApp's servers |
| Delivered | Delivered to the customer's device |
| Read | Customer has opened the message |
| Failed | Delivery failed — hover for the error reason |

If a message fails, the error reason from Meta is shown in the conversation. Common causes include an invalid phone number or the customer blocking your number.
